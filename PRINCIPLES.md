# InstaForge 제작 원칙

## 1. AI 파이프라인 설계 원칙

### 단계 분리 (Multi-step Pipeline)
하나의 거대한 프롬프트 대신 4단계로 분리하여 각 단계의 품질을 극대화했습니다.

```
콘텐츠 분석 → AIDA 구조 설계 → 카피라이팅 → 이미지 쿼리 생성
```

- 각 단계의 출력이 다음 단계의 입력이 되는 Chain-of-Thought 방식
- Structured Output (tool_use + Zod validation)으로 JSON 파싱 안정성 확보
- 카피라이팅 단계만 Opus 모델 사용, 나머지는 Sonnet으로 비용 효율화

### AIDA 프레임워크 적용
마케팅 검증된 AIDA(Attention-Interest-Desire-Action) 프레임워크를 슬라이드 구조에 적용하여 콘텐츠 완성도를 높였습니다.

### 프롬프트 엔지니어링 핵심 규칙
- 해요체 통일 (합니다체/반말 금지)
- 제목 8-15자, 본문 40-70자 제한
- CTA: 첫 슬라이드만 유도 문구, 중간은 비움, 마지막은 "저장해두세요" 고정
- 해시태그 정확히 4개, # 기호 미포함
- 캡션 200-400자, 이모지 2-3개

## 2. 디자인 품질 기준 (ralph-loop)

### 핵심 철학: "인스타 크리에이터 수준"
기술적 동작이 아니라 **결과물의 비주얼 퀄리티**가 핵심입니다.

### 비교 레퍼런스
| 계정 | 스타일 |
|------|--------|
| @toss.place | 풀블리드 사진 + 하단 큰 제목 + pill badge |
| @upzi____ | 초대형 bold 타이포 + 흰 배경 + 밈 감성 |
| @naver_official | 사진 위 강한 텍스트 오버레이 |
| @visualizevalue | 극한 미니멀 흑백 + 모노 타이포 |

### 디자인 감도 채점 기준 (0-10점)
| 항목 | 프로 | 아마추어 |
|------|------|----------|
| 사진 활용 | 풀블리드+선명, 사진이 주인공 | opacity 0.15, 배경 깔개 |
| 타이포 임팩트 | 72-88px, 900wt, 화면 지배 | 48px 이하 |
| 여백 | 캔버스 40%+ 비움, padding 80px+ | 꽉 찬 레이아웃 |
| 색상 절제 | 1-2색 + 흑/백 | gradient mesh, 5색+ |
| 장식 절제 | Glass card/corner accent 없음 | 장식 과다 |
| 인스타 느낌 | 잡지 표지 같음 | 대시보드 같음 |

**합격 기준: 모든 카테고리 8점 이상, CRITICAL FAIL 0건**

### 반복 검증 루프 (Ralph Loop v3)
디자인 품질을 보장하기 위해 자동화된 반복 검증 루프를 적용했습니다:
1. 시나리오별 캐러셀 생성 (트렌드 탐색, URL 입력, 이미지 변경, 컬러 팔레트)
2. JSON 품질 검증 (API 직접 호출)
3. Chrome 시각 검증 (스크린샷 기반)
4. 디자인 감도 채점 (5개 카테고리)
5. 미달 항목 수정
6. 반복 (모든 카테고리 8점 이상까지)

## 3. 이미지 매칭 원칙

### 멀티소스 전략
단일 이미지 소스의 한계를 극복하기 위해 4개 API를 병렬로 검색합니다:
- Unsplash (고품질 세로 사진)
- Pexels (다양한 세로 사진)
- Pixabay (넓은 커버리지)
- SerpAPI/Google Images (폴백)

### 관련성 스코어링
- 슬라이드 제목/본문의 키워드와 이미지 태그 간 매칭 점수 계산
- 기사 원문 이미지는 첫 슬라이드에 우선 배치
- 슬라이드 간 중복 방지 (globalUsed Set)
- 이미지 50% 미달 시 typo 템플릿으로 자동 전환

### 이미지 쿼리 규칙
- 2-4개 영어 명사만 사용 (동사 금지)
- 구체적 사물/장면 필수 포함
- AIDA 단계별 이미지 톤 차별화
- 인물/사물 이미지 교대 사용

## 4. UX 원칙

### 진행 상태 실시간 표시
SSE(Server-Sent Events)로 7단계 진행률을 실시간 스트리밍:
`분석 중 → 구조 설계 → 카피 작성 → 이미지 검색어 → 이미지 검색 → 렌더링 → 완료`

### 사용자 제어권 보장
- 이미지 자동 할당 후에도 수동 변경 가능
- 톤(친근/전문/위트), 슬라이드 수(4-12) 조절 가능
- 트렌드 탐색 / URL 직접 입력 두 가지 입력 방식

### Graceful Degradation
- 이미지 검색 실패 → 기사 이미지로 폴백
- 이미지 전체 부족 → typo 템플릿으로 자동 전환
- URL 접근 불가 → 명확한 에러 메시지

## 5. 코드 품질 원칙

### 타입 안전성
- Zod 스키마로 AI 출력 런타임 검증
- TypeScript strict mode
- API 요청/응답 전체 타입 정의

### 관심사 분리
```
src/
├── app/api/        # API 라우트 (서버)
├── lib/            # 비즈니스 로직 (claude, serpapi, validation)
├── hooks/          # 클라이언트 상태 (zustand store, pipeline)
├── components/     # UI 컴포넌트
└── templates/      # 캐러셀 디자인 템플릿
```

### 재시도 & 에러 핸들링
- Claude API 호출에 exponential backoff 재시도 (최대 2회)
- AbortController로 취소 가능한 요청
- 모든 외부 API에 10초 타임아웃
